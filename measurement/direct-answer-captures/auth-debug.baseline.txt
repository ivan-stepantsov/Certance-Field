The “exactly one hour” pattern is the tell: something is probably doing a local check like “issued at + 3600” or using a cookie/session `maxAge` of one hour, while the real session remains valid because the identity provider or backend has a longer or rolling lifetime.

The smallest plausible fix is to stop deriving expiry from a fixed one-hour constant and use the session’s authoritative expiry instead. In practice that usually means:

```ts
const expiresAtMs = session.exp * 1000; // JWT exp is seconds
if (Date.now() >= expiresAtMs) {
  return unauthorized();
}
```

not:

```ts
const expiresAtMs = session.iat * 1000 + 60 * 60 * 1000;
```
